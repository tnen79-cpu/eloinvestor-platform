# v49 Admin + Project Fixes

## المطلوب الذي تم تنفيذه

1. **طلب توثيق المشاريع يظهر في التوثيق**
   - عدلت Smart Form بحيث ينشئ طلب `verification_requests` للمشروع عند وجود طلب توثيق أو مستندات.
   - أضفت SQL يحوّل مستندات `project_documents` القديمة إلى طلبات توثيق ظاهرة للإدارة.

2. **نظام النشر مرتب أكثر**
   - أبقيت التحكم: نشر تلقائي / موافقة الإدارة.
   - SQL يدعم جدول `platform_settings` القديم الذي يستخدم `key/value`.

3. **مشاهدات المشروع فعالة**
   - أضفت API جديد: `/api/projects/view`.
   - `ProjectViewTracker` صار يرسل للـ API بدل الاعتماد على RPC من العميل فقط.
   - أضفت SQL لدالة Atomic: `increment_project_view_count` مع دعم text و uuid.

4. **القطاعات مرتبطة بلوحة التحكم**
   - الفورم يقرأ من `platform_sectors`.
   - SQL يضمن وجود جدول القطاعات وأعمدة التوافق.
   - أضفت قطاعات افتراضية إذا الجدول فارغ.

5. **إدارة المشاريع في لوحة الإدارة أنظف**
   - في قائمة المشاريع صار يظهر زرين فقط: فتح وإدارة / معاينة.
   - التحكم الكامل صار داخل نافذة المشروع: نشر، توثيق، تمييز، ترويج، طلب تعديل، رفض، حذف، تعديل البيانات والصور.

## ملف SQL المطلوب

