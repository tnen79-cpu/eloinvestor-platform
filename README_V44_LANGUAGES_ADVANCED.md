# v44 — تطوير نظام اللغات والترجمات

## الجديد

- SQL آمن يدعم جدول `platform_languages` القديم الذي يحتوي `name` إلزامي، والجديد الذي يستخدم `name_ar/name_en`.
- إصلاح خطأ `null value in column name` نهائياً عن طريق تعبئة `name` تلقائياً.
- حفظ اللغة من لوحة الإدارة يرسل `name` + `name_ar` + `name_en` حتى لا تتكرر مشاكل Supabase.
- زر ترجمة مبدئية داخل لوحة الإدارة للنصوص الأكثر استخداماً.
- زر تصدير JSON للنصوص الظاهرة في جدول الترجمات.
- جدول `translation_provider_settings` لتجهيز ربط مزود ترجمة لاحقاً بدون كشف المفاتيح في الواجهة.

## التشغيل

شغّل الملف:

```sql
SUPABASE_V44_LANGUAGES_ADVANCED_SAFE.sql
```

ثم افتح لوحة الإدارة > اللغات والترجمات.

## ملاحظة

الترجمة التلقائية الحالية مبدئية وآمنة ولا تستخدم API خارجي من المتصفح. عند ربط DeepL/Azure/Google لاحقاً يجب أن يتم ذلك عبر API route أو Supabase Edge Function حتى لا تنكشف المفاتيح.
